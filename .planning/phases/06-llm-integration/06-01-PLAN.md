# Phase 6 Plan: LLM Provider Integration

**Goal:** Multi-provider LLM support with token-efficient architecture

**Base:** v0.1 (all complete), Phase 6 for v0.2

## Plans

### Plan 6.1 — LLM Provider Abstraction
- `ethos_os/llm/base.py`: Base provider interface
- `ethos_os/llm/config.py`: Provider configuration
- Token-efficient: Abstract provider behind interface
- Streaming: Support streaming responses

### Plan 6.2 — Ollama Integration (Primary)
- `ethos_os/llm/providers/ollama.py`: Ollama provider
- Local-first: No API cost, privacy
- Default model: llama3
- Support: completion, embeddings

### Plan 6.3 — OpenAI + Anthropic + Azure
- `ethos_os/llm/providers/openai.py`: OpenAI provider
- `ethos_os/llm/providers/anthropic.py`: Anthropic provider
- `ethos_os/llm/providers/azure.py`: Azure OpenAI provider
- Fallback chain: Ollama → OpenAI → Anthropic

### Plan 6.4 — Agent Adapter Layer
- `ethos_os/agents/adapters/base.py`: Base adapter
- `ethos_os/agents/adapters/hermes.py`: Hermes adapter
- `ethos_os/agents/adapters/pi.py`: Pi adapter
- `ethos_os/agents/adapters/general.py`: General adapter

### Plan 6.5 — CEO Agent Integration
- `ethos_os/agents/chief_of_staff.py`: @chief-of-staff agent
- Strategic planning capabilities
- Program/Project creation from directives

## Success Criteria

1. All 4 providers usable via unified interface
2. Ollama works as primary (local)
3. Agent adapters wrap LLM calls
4. CEO Agent can execute strategic tasks
