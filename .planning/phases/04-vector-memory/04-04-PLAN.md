# Plan 4.4 — Memory ↔ Initiative Linking

**Goal:** Link vector memory to initiative hierarchy, invalidate on updates.

## Requirements
- MEM-04: Initiative docs chunked and embedded
- MEM-06: Cache invalidation on initiative updates

## Implementation

### Initiative ↔ Memory Linkage
- Each chunk stores `initiative_id` and `initiative_type`
- Initiative update triggers chunk re-indexing
- Delete initiative → delete all related chunks

### Automatic Indexing Triggers
1. **PRD Created/Updated** → Index PRD chunks
2. **Project Updated** → Index linked PRD
3. **Architecture Doc Uploaded** → Index sections
4. **Meeting Notes Added** → Index key decisions

### Cache Invalidation Strategy
```python
# On initiative update
def invalidate_initiative(initiative_id: str) -> None:
    # Clear working memory cache for initiative
    working_memory.cache_delete("initiative", initiative_id)
    
    # Re-index if content changed
    if content_changed:
        reindex_initiative(initiative_id)
```
