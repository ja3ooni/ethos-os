# Plan 4.1 — Qdrant Client and Collections

**Goal:** Qdrant integration with async client, collections per initiative level.

## Requirements
- MEM-01: Qdrant integration for initiative document chunking
- MEM-03: Context injection pipeline start

## Implementation

### Collection Strategy
Collections per initiative level:
- `initiatives_prd` — PRD chunks (intent, success_metric, scope, boundaries)
- `initiatives_docs` — Architecture docs, meeting notes
- `initiatives_conversations` — Chat message embeddings

### Collection Schema
```python
# Collection config
{
    "vectors": {
        "size": 384,  # MiniLM-L6-v2 dimensions
        "distance": "Cosine"
    },
    "optimize": {
        "indexed_vectors": 1024,
        "quantized_vectors": 2048
    }
}
```

### Metadata Fields
- `initiative_id` — Link to initiative (UUID)
- `initiative_type` — "portfolio|program|project|sprint|task|prd"
- `chunk_type` — "intent|scope|boundaries|meeting_notes|architecture"
- `created_at` — Timestamp
- `parent_chunk_id` — For chunk linking

### Files to Create
- `ethos_os/memory/qdrant_client.py` — Qdrant async client wrapper
- `ethos_os/memory/collections.py` — Collection management
