# Plan 4.3 — Context Injection to Agents

**Goal:** Semantic search → top-k chunks → working memory → agent prompt (token-efficient).

## Requirements
- MEM-03: Context injection pipeline
- MEM-05: Semantic search on initiative content

## Token Efficiency Principles
1. **Top-k retrieval**: Fetch only top 5 most relevant chunks
2. **Reference injection**: Pass chunk references, not full content
3. **Working memory cache**: Recent context in memory (not Qdrant)

## Implementation

### Context Injection Pipeline
```
User Query → Embed Query → Qdrant Semantic Search → Top-k Chunks → Working Memory → Agent Prompt
```

### Memory ↔ Qdrant Boundary
- **Qdrant**: Persistent semantic storage (documents, embeddings)
- **Working Memory**: Runtime context (recent chunks, agent state)

### Injection Formats

#### Semantic Search Request
```python
@dataclass
class SemanticSearchRequest:
    query: str
    initiative_id: str | None = None  # Filter by initiative
    initiative_type: str | None = None
    chunk_types: list[str] | None = None
    top_k: int = 5
    min_score: float = 0.7
```

#### Agent Context Response
```python
@dataclass
class AgentContextResponse:
    chunks: list[ChunkReference]  # Not full content
    total_retrieved: int
    tokens_estimate: int  # For monitoring
    cached: bool  # From working memory?
```

#### Chunk Reference (Token-Efficient)
```python
@dataclass
class ChunkReference:
    chunk_id: str
    initiative_id: str
    chunk_type: str
    summary: str  # 50-100 chars, not full content
    relevance_score: float
```
