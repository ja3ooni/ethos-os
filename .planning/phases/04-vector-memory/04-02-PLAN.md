# Plan 4.2 — Document Chunking and Indexing

**Goal:** Token-efficient chunking of initiative documents (512 tokens default).

## Requirements
- MEM-02: Working memory cache with TTL
- MEM-04: Initiative docs chunked and embedded

## Implementation

### Chunking Strategy
1. **PRD Chunking** (priority)
   - Intent: single chunk (~100 tokens)
   - Success metric: single chunk (~150 tokens)
   - Scope: chunked at 512 tokens
   - Boundaries: single chunk (~100 tokens)

2. **Document Types**
   - Architecture docs: section-based chunking
   - Meeting notes: key decisions extraction
   - Chat history: message clustering

### Token-Efficient Rules
- Target: 512 tokens per chunk (not characters)
- Overlap: 50 tokens between chunks (for context continuity)
- Max chunks per doc: 20 (10k tokens max per doc)

### Embedding Model
Using sentence-transformers (MiniLM-L6-v2):
- Dimensions: 384
- Speed: Fast inference
- License: Apache 2.0

### Files to Create
- `ethos_os/memory/chunker.py` — Document chunking logic
- `ethos_os/memory/embeddings.py` — Embedding generation
- `ethos_os/memory/indexer.py` — Indexing pipeline

### Indexing Flow
```
Initiative Update → Parse Documents → Chunk → Embed → Upload to Qdrant
```

## Verification
