# Plan 4.2 — Gate API

**Goal:** REST API for gate request creation and approval workflow.

**Tasks:**
1. Create `ethos_os/api/gates.py` with gate workflow endpoints
2. POST /gates — create gate request (manual trigger)
3. POST /gates/{id}/approve — approve gate with notes
4. POST /gates/{id}/reject — reject gate with notes
5. GET /gates/pending — list pending gates ordered by age
6. GET /gates/{entity_id}/history — gate history per entity
7. GET /gates/dashboard — dashboard data (counts, rates, aging)

**Dependencies:** Phase 2 gate models, GateRepository

**Agent:** @frontend-developer

**Success Criteria:**
- Gate workflow fully operable via API
- Pending gates queryable by age
- Gate history retrievable per entity
- Approval rate tracking works
