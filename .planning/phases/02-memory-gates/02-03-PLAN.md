# Plan 2.3 — Immutable Audit Log

**Goal:** Implement append-only episodic event log with hash-chain integrity.

**Requirements covered:** GATE-06, MEM-05, MEM-06

## Deliverables

### 2.3.1 — Audit Log Model
- `AuditLog` model: event_type, entity_id, entity_type, payload, hash, previous_hash, timestamp
- Event types: heartbeat, gate_decision, cost_snapshot, scope_change, budget_change
- Hash of previous record included in each record

### 2.3.2 — Hash Chain Integrity
- First record has null previous_hash
- Each subsequent record includes SHA-256 hash of (previous_hash + payload + timestamp)
- Verify integrity by re-computing hashes

### 2.3.3 — Append-Only Enforcement
- No update or delete operations on audit log
- Query by time range, entity, event type

## Implementation

- Create `ethos_os/models/audit.py` with AuditLog model
- Create `ethos_os/repositories/audit.py` with AuditRepository
- Implement hash chain generation and verification

## Acceptance Criteria

1. Audit log is append-only (no updates or deletes)
2. Hash-chain integrity maintained across records
3. Query by time range, entity, event type works
4. Integrity verification passes
