# Plan 2.2 — Gate Workflow

**Goal:** Implement auto-triggering and manual approval workflow for gates.

**Requirements covered:** GATE-01, GATE-02, GATE-03, GATE-04, GATE-05, GATE-07

## Deliverables

### 2.2.1 — Gate Trigger Service
- `check_scope_gate()`: Evaluates if task effort exceeds +25% threshold
- `check_budget_gate()`: Evaluates if actual cost exceeds +20% threshold
- Auto-create gate request when threshold exceeded

### 2.2.2 — Gate Workflow
- Gate blocking: task cannot proceed while pending
- Approver can approve or reject with notes
- Rejected task blocked until re-planned
- Configurable timeout (scope: 48h, budget: 24h)

### 2.2.3 — Gate Repository
- `create_gate_request()`: Create new gate request
- `approve_gate()`: Approve gate with optional notes
- `reject_gate()`: Reject gate with optional notes
- `get_pending_gates()`: List pending gates
- `get_gates_for_entity()`: Get all gates for an entity

## Implementation

- Create `ethos_os/repositories/gate.py` with GateRepository
- Implement trigger evaluation logic
- Implement approval/rejection workflow

## Acceptance Criteria

1. Scope gate auto-created at +25% estimate variance
2. Budget gate auto-created at +20% cost variance
3. Rejected task remains blocked until re-planned
4. Approved gate unblocks task
5. Configurable timeouts work correctly
