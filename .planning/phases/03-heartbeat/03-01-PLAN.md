# Plan 3.1 — Heartbeat Scheduler

**Goal:** Heartbeat scheduler with configurable intervals and status tracking.

**Requirements:** BEAT-01, BEAT-02, BEAT-04

**Assignment:** @backend-architect

## Implementation

### 3.1.1 Agent Registry Model

Create `ethos_os/execution/agent.py`:
- Agent model with id, name, agent_type, status, heartbeat_interval, last_heartbeat_at
- Status enum: idle, working, blocked, complete
- Configurable heartbeat_interval (default: 30s, min: 10s)

### 3.1.2 Heartbeat Record Model

Create `ethos_os/execution/heartbeat.py`:
- Heartbeat record: timestamp, agent_id, status, task_id, progress_note
- Status enum matching agent status

### 3.1.3 Scheduler Implementation

Create `ethos_os/execution/scheduler.py`:
- asyncio-based heartbeat scheduler
- Interval management per agent
- Status variance (don't log idle repeatedly)

## Acceptance Criteria

- [ ] Agent can register with configurable heartbeat interval
- [ ] Heartbeat recorded on correct interval (default 30s)
- [ ] Heartbeat record includes: timestamp, agent_id, status, task_id, progress_note
- [ ] Status variance implemented (idle heartbeats only when status changes)
- [ ] Min interval enforced (10s)

## Files

- `ethos_os/execution/agent.py` — Agent registry model
- `ethos_os/execution/heartbeat.py` — Heartbeat record model
- `ethos_os/execution/scheduler.py` — Heartbeat scheduler
