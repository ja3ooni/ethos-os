# Plan 3.2 — Agent Executor Loop

**Goal:** Gate-aware agent execution loop with episodic logging.

**Requirements:** BEAT-03, BEAT-06

**Assignment:** @backend-architect

## Implementation

### 3.2.1 Executor Loop

Create `ethos_os/execution/executor.py`:
- Per heartbeat: check assigned tasks → check gate status → execute eligible → update working memory → write episodic log → report
- Gate-aware: cannot execute gated work (blocked by pending gate)
- Working memory update per execution cycle

### 3.2.2 Gate-Aware Execution

Integrate gate checks:
- Query pending gates for assigned tasks before execution
- Skip tasks with pending gates

### 3.2.3 Working Memory Integration

Update working memory on each execution cycle:
- Current task
- Reasoning state
- Progress notes

### 3.2.4 Episodic Log Integration

Write heartbeat to audit log on each cycle:
- Event type: heartbeat
- Entity: agent
- Payload: status, task_id, progress_note

## Acceptance Criteria

- [ ] Executor checks assigned tasks on each heartbeat
- [ ] Gate status checked before task execution
- [ ] Tasks with pending gates are skipped
- [ ] Working memory updated each cycle
- [ ] Episodic log written on each heartbeat
