## Key Implementation Decisions (D-*)

### D-04 to D-08: Materialized Path
- Each entity stores: `parent_id`, `path`, `path_depth`
- Path format: `/uuid1/uuid2/...`
- Lineage query: O(1) via path split

### D-09 to D-12: Repository Pattern
- SQLAlchemy 2.x with DeclarativeBase
- `BaseRepository` with generic CRUD
- Pydantic models deferred to API phase

### D-13 to D-17: Working Memory
- Dict keyed by `agent_id`
- `threading.Lock` for thread safety
- TTL-based pruning (default 1 hour)

### D-20 to D-22: Entity Fields
- Project: `prd_status` enum (draft → pending_approval → approved → archived)
- Sprint: `goal`, `start_date`, `end_date`, `capacity_hours`
- Task: `prd_scope_item_id` required

### D-44 to D-45: Validation
- Task creation: raises ValueError if `prd_scope_item_id` is null
- Sprint creation: raises ValueError if project.prd_status != approved
